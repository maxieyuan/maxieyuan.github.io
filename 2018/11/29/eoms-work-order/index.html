派发EOMS工单操作说明

置顶 | 发表于 2018-11-29 | 分类于 EOMS | | 阅读次数: |
字数统计: 934

在代维人员日常使用网络代维系统时，如果遇到操作失误、修改信息、提取数据的需求时，就需要在EOMS系统上派发工单到代维系统维护人员的账号下，由维护人员进行处理。

对于问题的重要程度，可以根据派发工单时的紧急程度进行选择。

请各位代维人员以及需要派发工单解决问题的人员注意以下几点！

  1. 处理问题前，如已知该问题需要派工单，请先派EOMS工单，在工单中对需要处理的问题进行描述，必要时可以派单后，电话与代维系统维护人员进行沟通。
  2. 处理问题前，如不清楚该类问题是否需要派单，请电话联系代维系统维护人员，确认是否要派发工单。如需要派发工单，则先派发工单，待工单流转到代维系统维护人员账号下之后进行处理问题。
  3. 对于紧急的问题，需要维护人员加急处理的，请先派发EOMS工单，并将EOMS工单号复制给代维系统维护人员进行确认，确认无误后，进行处理问题。
  4. 派发工单时，请阅读下面编写的派发EOMS工单手册。如未按照手册进行派发EOMS工单，则会被审核人驳回；如未阅读以下手册，并且致电代维系统维护人员，询问如何派发EOMS工单，代维系统维护人员有权不予解答。
  5. 对于未派发工单，隐瞒自身操作失误，且认为是系统原因造成问题的代维人员，经过维护人员对该类人员提出问题检查后发现为人为操作失误导致，网络代维系统有权对该类人员进行信用评级，降级该类人员的处理及时性。
  6. 派发EOMS工单只有各个地市的移动人员才有权限，代维人员无权限进行派发EOMS工单，这一点请务必知悉！

¶派发EOMS工单

内蒙古网络代维系统属于网管系统维护流程中的系统，所以需要在EOMS流程管理中的网管系统维护流程进行派发工单

新增工单时需要注意的事项：

  1. 工单主题：描述哪个地市，遇到什么问题。主题内容中请避免删除、修改数据等字样
  2. 专业：网管
  3. 系统：代维管理系统
  4. 维护分类：服务类
  5. 问题发现世界：根据实际时间，进行选择即可
  6. 合同名称：亿阳服务合同
  7. 合同服务内容：根据实际填写，一般处理问题为配合日常工作即可
  8. 问题描述：将需要处理的问题进行描述
  9. 附件：如果问题描述不能说明问题的情况，请上传附件
  10. 选择派发对象：代维管理分析组-网管支撑室
  11. 紧急程度：
    • 紧急：流转到代维系统维护人员账号下需要两个工作日，处理完成在工作日的12个小时之内。
    • 重要：流转到代维系统维护人员账号下需要四个工作日，处理完成在三个工作日之内。
    • 一般：流转到代维系统维护人员账号下需要十五个工作日，处理完成在十二个工作日之内。
